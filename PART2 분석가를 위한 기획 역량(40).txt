PART2 분석가를 위한 기획 역량(40)

1. 디자인씽킹?
객관식 정도 나오지 않을까..? 이거 뭔지 설명하라고 서술형 나오진 않...겠지

ex) 소비자들이 가치있게 평가하고 시장의 기회를 이용할 수 있으며, 
      기술적으로 가능한 비즈니스 전략에 대한 요구를 충족시키기 위해
       디자이너의 감수성과 작업방식을 이용한 문제해결을 위한 사고방식을 뭐라고 하게?

ex) 디자인씽킹 특성이 아닌 것은?
공감 / 통합적 사고 / 개인의 능력 / 낙관주의 / 실험주의 / 협업 

2. Ux
뭔가 걍 기사시험에서 틀렸던건데 눈에 밝힘,, 그리고 일단 이거도 얘기하긴 했어서,,,
사용자 경험 중심, 경험 중심 디자인, 이성적이면서 감성적, 대량 맞춤형, 등

3. 브레인스토밍
객관식 가능성 좀 있음 + 주관식 가능성 높음

ex) 브레인 스토밍에 대해서 써봐라~
아이디어가 많을수록 좋은 아이디어가 나올 가능성이 높기 때문에
비판을 금지하고, 자유롭게 많은 아이디어를 추구하는 디자인씽킹 프로세스

4. 프로토타입
객관식이든 주관식이든 나올지는 모르겠네
시제품, 신속하게 만들어 평가, 시제품에 집작하지 않음,
처음부터 개량될 여지가 있는 아이디어는 별로 좋은 아이디어가 아님

5. 사용자 여정지도
객관식이든 주관식이든 나올지는 모르겠네
고객의 일련의 행동 과정을 시간축으로 가시화한 그래프

6. 페르소나
객관식이든 주관식이든 나올지는 모르겠네
제품이나 서비스를 사용할 사용자 유형을 대표하는 가상인물

7. biz-driven / data-driven
이거 좀 강조했었는데 어떻게 나올지..는 모르겠음
실직적이고 실무적인 빅데이터 과제 도출의 접근방법임

biz-driven
top - down 방식,  business value chain 분석,
브레인스토밍 / 워크샵 / 마인드맵 등을 통해 주요 업무 프로세스 별 개선요소를 분석함

data-driven
bottom - up 방식, 내외부 수요과제,
인터뷰, 설문조사 등을 통해 민원 요구사항이나 부서의 수요를 도출해냄

8. MECE / LISS
이거도 얘기 두번인가 세번 했었는데 나올지..는 모르겠음 내기엔 어렵지 않나?

MECE : 뇌피셜, 상호배제, 전체포괄, 계획 수립
LISS : 데피셜, 상호배타, 핵심만 나열, 문제 해결

9. 빅데이터 분석
이거도 분석자 엔지니어 과학자 얘기하면서 조금 얘기했었는데 나올지..는 모르겠음

분석 부하를 줄이기 위해 분석변수를 제한적으로 활용함
다양한 기법이 있음
데이터 간 인과관계를 먼저 분석한 다음 비즈니스적 의미를 발견함
기술에 대한 학습 + 훈련으로 HARD SKILL을 + 고객 등과의 커뮤니케이션을 통해 SOFT SKILL을 익혀야 함

10. 빅데이터 분석 방법론
절차와 방법 강조하긴 했음 특히 314p 별5개 쳐져있던데

10.1 방법론과 빅데이터 분석 방법론
오 그래서 방법론이 뭔데?
이론적으로 문제를 처리하기 위해 체계적인 절차와 방법을 정리한 것임
즉, 빅데이터 분석 방법론이 뭐냐?
빅데이터 분석을 효과적으로 수행하기 위한 체계적인 절차와 방법을 정리한 것임

10.2 문제를 해결하는 방법
Top-down 방식을 통해 가설 수립하고 검증함
그리고 Bottom-up 방식으로 데이터 분석하고 패턴 확인함
Optimization : 최적배치모델
Insight : 과거경험을 기반으로 현황을 분석함

10.3 방법론 선택
KDD / SEMMA / CRISP-DM
개인적으로 해보라고 했지만 일단 워크샵1에 있던건 KDD였음 
솔직히 진짜로 안 나올거 같긴한데 이거 얘기 좀 했었음

KDD
insight 발굴을 위한 절차와 단계를 정의한 데이터 마이닝 기법
일반적으로 아는 분석 단계임
데이터 선택 - 데이터전처리 - 데이터셋변환 - 모델링 - 평가 이런거

SEMMA
데이터 탐색 5개 프로세스 기반 데이터 마이닝 기법

CRISP-DM
비즈니스와 데이터 이해를 시작으로 6단계로 수행되는 데이터마이닝 방법론
공공기관에서 많이 사용한대

11. 데이터 시각화
객관식가능성 조금 + 이건 진짜로 강조 많이 했기 때문에 서술형 가능성 높음
좋은 빅데이터 시각화는 분석을 가시화하여 인사이트를 쉽게 얻을 수 있도록 하는 것임

ex) 데이터 시각화에 대해 써봐라
방대한 양의 자료를 분석해서 한눈에 볼 수 있도록 도표나 차트 등으로 정리한 것

ex) 데이터 시각화로 얻을 수 있는 효과?
자료로부터 정보를 습득하는 시간 절감
흥미 유발 및 정보의 빠른 확산 촉진
빠르게 자료를 기억하는데 도움

ex) 데이터 시각화 단계?
분석  -> 선별 -> 마이닝 -> 표현

12. 탐색적 문제해결(EDA)
객관식 가능성 조금 있음 + 주관식 가능성 있음
이거도 앞에서부터 계속 얘기했던건데 어떻게 나올지는 모르겠네..

ex) 데이터 분석을 위한 데이터 패턴이나 주요 변수의 특징을 확인하는 단계를 뭐라고 하게?

13. 인포그래픽
데이터 시각화랑은 조금 다름
우리가 알고있는 정보를 남에게 전달하기 위해 정보를 조금 더 과장해서 기억에 남도록 보여주는 것
